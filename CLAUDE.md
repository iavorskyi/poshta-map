# Проєкт: poshta-map

Сервіс для рутинних задач листоноші (пенсіонери, виплати, обходи, організації, передплати).

**Повний опис архітектури, моделі даних і фіч — у [docs/APP_OVERVIEW.md](docs/APP_OVERVIEW.md). Це джерело правди.**

## Стек
- Next.js (App Router) + Prisma 6 + **PostgreSQL** (спільна віддалена база, `DATABASE_URL` у env).
- Є логін/ролі: `requireUser()` у `src/lib/auth.ts`, права у `src/lib/permissions.ts`.

## Робота зі схемою БД
- Міграційних файлів немає — схема застосовується через `npx prisma db push` (білд робить `prisma db push --skip-generate`).
- **УВАГА:** `db push` б'є одразу по спільній проді-базі. Зважай на це перед зміною схеми.
- Глобальні key-value налаштування — у моделі `AppSetting`.

## Деплой
- Ручний: `vercel --prod` з **головного** worktree. GitHub auto-deploy НЕ налаштований. Не запускати `vercel` з worktree.

## Конвенції
- Після бізнес-логічних змін одразу оновлювати `docs/APP_OVERVIEW.md` у тому ж комміті.
- Форми — інлайн, без модалок (див. приклади в секції організацій).
